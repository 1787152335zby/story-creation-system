# 角色形象分组与变体识别设计

## 问题

生图需求清单和角色列表各自平铺展示所有角色条目，基础形象和变体混在一起，用户分辨困难。

测试61示例：7 个角色平铺（林深 / 苏晚 / 地上的"林深" / 系统【盖亚】 / 林深（画面中） / 系统【盖亚】（画外音） / 林深_受伤形象），其中 3 个是伪角色（纯镜头位置描述），1 个是真变体。

## 目标

1. 生图清单按 `character_base` 分组，基础形象默认展示，变体折叠在下
2. 一键生成分两步：先所有基础，再所有变体
3. 伪角色（位置/镜头标记）在 `image_preparator` 阶段归并，不回标到清单

## 类型判定规则

| 类型 | 条件 | 实现阶段 |
|------|------|:---:|
| 基础形象 | `character_base` 指向自己，`is_base: true` | 数据层 |
| 变体 | `character_base` 指向他人，`is_base: false` | 数据层 |
| 伪角色 | 角色名为"已知角色 + 纯位置/镜头标记词" | 本次实现在 `image_preparator` 阶段过滤

### 伪角色归并关键词

位置/镜头标记词：`地上、画中、画面中、画面内、画外、画外音、远景、近景、中景、特写、背面、正面、侧面、仰视、俯视`

处理流程：在 `_parse_shots` 收集角色时，对每个角色名做如下判断——

1. 若角色名已在 `all_char_states` 中 → 保留
2. 若角色名去掉所有括号、引号、空格后，与已知基础角色名完全相同 → 映射为基础角色
3. 否则保持原样（可能是真正的新角色，如"系统【盖亚】"）

## 设计

### 1. 后端：`image_preparator.py` — `_parse_shots` 增加伪角色归并

```
在收集 all_char_states 阶段：
  for char_name in shot.characters:
    1. 过滤"无"/"无人"/"空"
    2. 匹配伪角色关键词 → 映射为基础角色名
    3. 对已知基础角色无增益的，镜头计数归基础角色
```

### 2. 后端：`projects.py` — `get_image_demands` 返回分组结构

新增字段 `character_groups`：

```json
{
  "character_groups": [
    {
      "name": "林深",
      "total_shots": 140,
      "members": [
        { "name": "林深", "is_base": true, "shots": 140, "variant_name": "基础形象" },
        { "name": "林深_受伤形象", "is_base": false, "shots": 140, "variant_name": "受伤形象" }
      ]
    }
  ]
}
```

向后兼容：保留原有 `characters` 扁平数组。

### 3. 前端：`ProjectImageGenForm.tsx` — 生图需求清单分组渲染

```
📋 生图需求清单
┌──────────────────────────────┐
│ 林深                 140镜 ▾ │  ← 点击展开/折叠
│   基础形象    140镜 ✓       │  ← 已确认则绿勾
│   受伤形象    140镜 ⚠       │  ← 变体，灰显
│                              │
│ 苏晚                  15镜 ✓│  ← 无变体，直接显示
│ 系统【盖亚】           9镜   │
└──────────────────────────────┘
```

- 每个 group 的 `members[0]` 是基础形象，其余是变体
- 变体行在 `_confirmed` 未标时显示灰色 ⚠
- 一键生成按钮读 `members` 而非 `characters`

### 4. 前端：一键生成分两步

```
[⚡ 第一步：生成所有基础形象 (3个)]
[⚡ 第二步：生成所有变体 (1个)]  ← 灰显直到基础全确认
```

步骤 1：遍历所有 group，取 `members.filter(m => m.is_base)`，逐个生成
步骤 2：遍历所有 group，取 `members.filter(m => !m.is_base)`，逐个生成（变体自动以基础角色已确认图为参考）

### 5. 前端：角色列表保持一致分组

保持现有 `charTree` (`character_base` 父子分组)，不额外改动。若视觉圣经中也有伪角色（如"林深（地上）"），按相同规则在 `_load_or_extract_visual_bible` 阶段处理。

## 不变的部分

- 场景列表渲染逻辑不变
- 道具列表不变
- 单角色/场景选中生成流程不变
- `handleCharSelect` 自动参考图逻辑不变

## 不在本次范围

- 视觉圣经抽取算法的改进（`VisualBibleExtractor` 阶段正确标记 `character_base`）—— 后续单独处理，本次专注于前端展示和后端数据输出的改造。
- 伪角色在视觉圣经 JSON 文件中的清理 —— 本次仅在 `image_preparator` 流水线中过滤，原文件不改动。

## 验收

1. 测试61 生图清单从 7 个平铺条目变为 3 个分组（林深/苏晚/系统盖亚）
2. 林深组展开显示基础+受伤两个子项
3. 一键生成按钮显示「第一步：生成所有基础形象 (3个)」
4. 基础全确认后「第二步：生成所有变体 (1个)」可用
5. 伪角色（地上的"林深"等）不出现在清单中，镜头数归入所属基础角色
