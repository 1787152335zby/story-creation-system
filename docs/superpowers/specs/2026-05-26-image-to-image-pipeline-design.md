# 图生图（Image-to-Image）管线设计

## 1. 背景与目标

### 1.1 问题现状
当前系统仅支持纯文生图（text_to_image）模式，所有资产都是从零开始生成。存在以下限制：
- 无法基于已有角色/场景/道具进行修改（如换衣服、改细节）
- 没有利用已确认的基础形象作为参考
- 变体生成只能通过重新编写提示词，缺乏一致性

### 1.2 目标
在现有文生图管线基础上，增加图生图（image_to_image）能力，实现：
- 保留文生图模式（从零生成基础形象）
- 新增图生图模式（基于参考图修改）
- 与现有资产系统无缝集成
- 支持变体管理和新资产创建

## 2. 核心概念

### 2.1 两个模式

| 模式 | 说明 | 用途 |
|------|------|------|
| **文生图** | 纯提示词生成全新资产 | 创建基础形象、新项目初始化 |
| **图生图** | 参考图 + 提示词修改 | 变体设计、服装更换、细节调整 |

### 2.2 保存策略

| 策略 | 说明 | 适用场景 |
|------|------|----------|
| **存为变体** | v1→v2→v3... 版本链 | 同一角色的不同尝试、迭代优化 |
| **存为新资产** | 独立新文件 | 完全不同的新设计、衍生角色 |

## 4. API 设计

### 4.1 图生图端点：`POST /assets/modify`

**请求：**
```json
{
  "project_name": "我的故事",
  "asset_type": "角色",
  "asset_name": "张三",
  "variant": "基础形象",
  "prompt": "换成红色连衣裙",
  "negative_prompt": "",
  "strength": 0.7,
  "model": "image-2",
  "save_as": "variant",
  "new_asset_name": ""
}
```

**字段说明：**
- `asset_type`: "角色" | "场景" | "道具"
- `variant`: 可选，默认用基础形象
- `strength`: 0-1，越高越偏离原图
- `model`: 可选，切换生图模型
- `save_as`: "variant" | "new_asset"
- `new_asset_name`: `save_as=new_asset` 时必填

**响应：**
```json
{
  "status": "ok",
  "path": "/07_生成素材/角色/张三/v2/基础形象.png",
  "version": 2,
  "url": "http://localhost:8000/.../v2/基础形象.png"
}
```

## 5. 目录结构与版本管理


## 6. 错误处理

| 场景 | HTTP 状态码 | 错误信息 |
|------|-------------|----------|
| 原图不存在 | 404 | "参考资产不存在，请先生成基础形象" |
| API 调用失败 | 500 | "图生图 API 调用失败：{error}" |
| strength 超出范围 | 400 | "strength 必须在 0-1 之间" |
| 新资产名冲突 | 400 | "资产名已存在，请使用其他名称" |
