# 文本生成管线优化 — 阶段一 & 阶段二

## 总览

优化故事创作流水线的前两个阶段，解决版本选择丢失、质量无检查、分块切割粗糙、内容量失控等核心问题。不改变整体架构，只增强提示词策略和后处理校验。

---

## 阶段一：故事大纲

### 当前流程

```
用户输入创意 → AI 生成版本A + 版本B（完整大纲）→ 用户选择 → 清理未选版本
```

### 问题
1. 没有轻量级方向确认环节：上来就出完整大纲，如果方向不对全白费
2. "经典" vs "反套路"区分不够明显，用户难以选择
3. 角色设定缺少关系张力
4. 没有结构可视化

### 改进方案

**改为两轮制：**

#### 第一轮：故事方向卡

轻量级输出（300 字以内），包含：

```
【一句话梗概】XX 字的 XX 题材故事，关于……
【视角人物】主角是谁，TA 想要什么
【核心冲突】阻挡 TA 的最大障碍
【情绪基调】冷峻/温暖/悬疑/热血……
【类型对标】类似《XXX》的风格方向
【两个方向选项】
  方向A:（一句话说明走向）
  方向B:（一句话说明走向）
```

用户确认方向卡后，再进入完整大纲生成。

**实现方式：** 在 `outline_designer.txt` prompt 中增加新指令——先输出方向卡，等待确认后再继续。orchestrator 处增加一个确认步骤。

#### 第二轮：完整大纲 + 角色设定

保留现有机制。角色设定增加「关系张力」字段：每个角色对其他角色的核心情感和冲突点。

## 阶段二：完整剧情

### 当前流程

```
大纲被选中版本 → 按故事类型分块（三幕/多集/自动） → 逐块生成 → 每块提
取摘要 → 合并输出
```

### 问题与解决方案

#### 2.1 版本选择信息丢失

**问题：** `_cleanse_outline_version` 只删掉了未选版本文本，没把"为什么选这个版本"的信息传下去。后续阶段不知道方向偏好。

**方案：**
- `_cleanse_outline_version` 清理后，AI 自动分析两个版本的结构差异
- 在文件末尾追加 `✅ 已选中版本A/B。差异摘要：{AI 自动总结}`
- 剧情阶段读大纲时，这个差异摘要作为 `{confirmed_direction}` 传入 prompt

#### 2.2 质量门禁无检查机制

**问题：** prompt 里有 6 条质量门禁，但 AI 写了不等于遵守了。没有检查手段。

**方案：** 生成完后 AI 自检。

自检步骤：
1. 剧情生成完毕后，AI 再次读取自己的输出
2. 逐条对照 6 条质量门禁：
   - 禁止前重后轻 → 检查前 30% 和后 30% 的字数比
   - 禁止过渡跳跃 → 检查相邻场次之间是否有衔接
   - 配角不是工具人 → 检查每个配角是否至少有一段"非信息传递"描写
   - 关键反转需要铺垫 → 检查反转前是否有伏笔
   - 最终反派的重量 → 检查反派出场篇幅
   - 每场一句记忆点 → 检查每场是否有金句或记忆点
3. 输出检查报告：`【质量自检】5/6 通过 · 问题：第7-8场过渡跳跃`

如果通过率低于 4/6，自动提示用户是否要重写。

#### 2.3 分块切割割裂剧情流

**问题：** 当大纲中没有找到明确的章节标记时，退化为"按字数均分"，导致段落被切断，前后衔接断裂。

**方案：** 所有分块方式统一加上"预分析分割点"步骤。

预分析步骤：
1. AI 读取完整大纲
2. AI 判断"这个故事最适合在哪几个位置切分"（基于事件断点、幕结构、场景转换）
3. 输出分割点列表，传入 ChunkIter

对于"自动（0 chunks）"模式（短剧/电视剧/小说）：
- 让 AI 先用一句话判断章节数：`请判断应该分几集，只输出一个整数`
- 然后针对每集让 AI 标注"本集覆盖大纲的哪部分内容"
- 把标注结果传给剧情生成

#### 2.4 内容量无后处理校验

**问题：** prompt 里有场次/字数约束表，但 AI 经常超量，生成后没有检查。

**方案：** 生成后统计 + 超限提示。

校验步骤：
1. 统计总字数、总场次数
2. 对照约束表计算偏差百分比
3. 输出：`【内容量】目标 2000-4000 字，实际 6200 字（超 55%）`
4. 如果超限超过 30%，提示用户：
   - 确认是否精简
   - 或者调整时长设定

精简方式：让 AI 按场景缩减，每场保留核心冲突和关键对白，裁剪修饰性描写。不做整体重写。

#### 2.5 关键元素缺乏完整性追踪

**问题：** 摘要只追踪"已经写了什么"，不核对"大纲答应要写什么"。

**方案：** 承诺清单。

实现步骤：
1. 剧情生成前，AI 从大纲中提取「承诺清单」：
   - 必须出场的角色
   - 必须发生的关键事件
   - 必须解决的核心冲突
2. 承诺清单作为文件头写入剧情文件
3. 剧情生成后，AI 逐项检查承诺是否兑现
4. 未兑现项标注在检查报告中

### 保留的现有机制

- `reverse_order = True`（先写后幕再写前幕，用于伏笔埋设）：保留
- `summarize = True`（每块写完后提取摘要）：保留，同时增加承诺清单对比

---

#### 2.6 故事质量深度提升：潜文本、角色声音、节奏、呼吸感

**背景：** 剧本阶段的问题根因在剧情阶段——潜文本、角色声音、节奏控制、呼吸感这些故事质量层面的事，在剧本阶段无法补回来。必须在剧情阶段解决。

**新增 4 项改进：**

##### 2.6.1 潜文本（Subtext）

**问题：** AI 写的对白和叙述过于直白，角色想到什么说什么，缺少"表面意思 vs 真实意思"的张力。

**方案：** 在每个场景生成前，增加"潜文本分析"步骤。AI 先自问两个问题，然后带着这个理解去写：

```
【潜文本分析 — 场景生成前必做】
对当前场景，先回答：
- 表面发生了什么？（角色A说了什么、做了什么）
- 实际在发生什么？（角色A真正的意图、隐瞒了什么、感受到了什么没有说）
把这个分析作为理解场景的深层背景，写入叙述时体现这种张力。
```

通过 prompt 指令实现，不需要改动代码结构。

##### 2.6.2 角色声音（Character Voice）

**问题：** 所有角色说话风格趋同，无法区分。

**方案：** 在剧情生成前，从大纲的角色设定中提取"角色声音卡片"。写入每块生成的 prompt 中。

角色声音卡片格式：
```
### 角色声音参考
各角色的说话风格请参考以下特点（不是逐句照搬，而是把握感觉）：

林深：刑警，35岁。短句为主，喜欢反问。语调冷静但有压迫感。
       常用词汇："有意思。""继续。""你确定？"
       语速偏慢，会在句中停顿。

林雪：记者，28岁。话多，语速快，喜欢用比喻。
       常用词汇："你听我说……""等一下，那不对"
       偶尔自嘲，紧张时会说冷笑话。
```

**实现方式：** outline_designer 阶段输出大纲时附带角色设定信息。`plot_expander` 在生成前从大纲文件中解析角色设定，按固定格式提取角色声音特征，插入 prompt。

##### 2.6.3 节奏控制（Rhythm）

**问题：** AI 的输出叙事节奏均匀，没有用句长/段长控制紧张与舒缓。

**方案：** 在 prompt 中增加节奏规则表，按场景类型控制段落格式：

```
【节奏控制规则】
不同场景类型请按以下段落格式输出，控制阅读节奏：

| 场景类型 | 段落长度 | 句长 | 效果 |
|---------|---------|------|------|
| 打斗/追逐/爆炸 | 1-2句一段 | 短句（8-15字） | 急促、紧张 |
| 对话/交锋 | 3-5句一段 | 中长句交替 | 有呼吸感 |
| 情绪高潮 | 2-3句一段 | 中句为主 | 聚焦情感 |
| 氛围/环境建立 | 长段展开 | 长句为主 | 舒缓、沉浸 |
| 沉默/留白 | 短段 + 空行 | 极短句或无句 | 制造停顿 |
```

##### 2.6.4 呼吸感（Breathing Room）

**问题：** AI 倾向于每个场景都塞满事件，没有"空气"。

**方案：** 在 prompt 中增加规则：

```
【呼吸感规则】
- 每 3-5 场连续的事件推进场景后，必须有一场"呼吸场景"
- 呼吸场景的特征：没有新事件发生、没有关键信息传递
- 可以是：角色独处、环境空镜、沉默的旅程、等待
- 呼吸场景的目的是让读者/观众消化前面积累的情绪
- 示例：追逐战后，主角坐在天台上看着城市灯光，一句话没说
```

##### 实现方式

所有 4 项改进均通过修改 `prompts/plot_expander.txt` 实现，不涉及代码改动。在 prompt 中增加对应的指令段落即可。

---

## 阶段三：完整剧本（定位修正）

### 当前定位

阶段三只做技术性转换和格式校验，不做创作性改进。创作性质量（潜文本、角色声音、节奏、呼吸感）在阶段二解决。

### 待改进项（后续实施）

1. 字数硬约束：生成前写入字数上限，生成后校验
2. 场次 ID 继承：从剧情文件中继承场次编号
3. 对白自然度检查：生成后自检
4. 方向传递：从剧情文件读取风格信息
5. 生成后校验：字数/场次覆盖/对白质量

## 实施范围

| 文件 | 改动 |
|------|------|
| `agents/outline_designer.py` | 增加方向卡输出逻辑 |
| `agents/orchestrator.py` | 增加方向卡确认步骤；`_cleanse_outline_version` 增加差异总结；增加承诺清单提取和自检调用 |
| `agents/plot_expander.py` | 接收 `{confirmed_direction}` 参数；生成后调用自检；增加字数统计和超限校验 |
| `prompts/outline_designer.txt` | 增加方向卡指令 |
| `prompts/plot_expander.txt` | 增加版本方向信息、质量自检指令、承诺清单指令 |
| `core/chunk_strategy.py` | 增加预分析分割点方法 |

## 不涉及

- 前端界面（不新增 UI 组件）
- 数据库表结构
- API 接口
- 其他 4 个阶段（剧本/分镜/视觉提取/提示词）
