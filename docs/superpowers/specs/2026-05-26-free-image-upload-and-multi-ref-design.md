# 图生图自由上传区 + 多图引用 + 双模式协作 — 设计文档

## 1. 背景与目标

### 1.1 问题
当前生图页面的图片参考被限制在 4 个分类（画风/人物/场景/道具）中，用户无法：
- 自由上传任意图片作为图改图的底图
- 同时使用多张参考图（如角色图 + 衣服图）
- 在提示词中用 @引用 精确指定每张图的用途

### 1.2 目标
在生图页面新增双模式自由上传区：
- **模式1（参考图生图）**：自由上传图片作为底图，支持多图 + @引用机制
- **模式2（风格参考）**：保留现有 4 分类参考体系，增强为真正影响生成风格的参考
- **双模式可同时使用**：真人底图 × 动漫风格参考 × "换成红裙子" = 动漫风红裙角色

## 2. 前端设计

### 2.2 @引用机制

**触发方式：**
- 提示词输入框右侧有 `@` 按钮
- 提示词中输入 `@` 自动弹出图片选择器
- 模式1 区域底部有 @图N 快捷按钮

**显示效果：**
- 提示词中 `@图1` 显示为绿色高亮标签
- 每个标签可点击移除

**后端处理：**
- 前端发送 `prompt` 中的 `@图N` 原文 + `reference_images` 数组
- 后端将 `@图N` 替换为对应 base64，或作为多图送入 API

### 2.3 多图上传

- 拖入多张图自动编号：图1、图2、图3...
- 单张图可删除
- 最大数量由所选模型的能力决定

## 3. 后端设计

### 3.1 ImageBackend 能力标志

```python
class ImageBackend(ABC):
    # 新增
    MAX_REFERENCE_IMAGES: int = 1
    SUPPORTS_IMG2IMG: bool = True
```

| 后端 | MAX_REFERENCE_IMAGES | 说明 |
|------|---------------------|------|
| SeedreamBackend | 1 | 标准图生图 |
| OpenAICompatBackend | 1 | 聚合平台通用 |
| 未来可扩展 | N | Gemini 等支持多图 |

### 3.2 image_to_image 签名调整

```python
def image_to_image(
    self,
    images_base64: list[str],  # 改为列表
    prompt: str,
    ...
) -> list[str]:
```

- 如果模型只支持 1 张，取 `images_base64[0]`
- 如果模型支持多张，全部传入

### 3.3 模型列表 API 扩展

`/settings/models` 返回的模型对象增加能力字段：

```json
{
  "value": "doubao-seedream-4-5-251128",
  "label": "Seedream 4.5",
  "max_ref_images": 1,
  "supports_img2img": true
}
```

## 4. 能力不足时的提示

前端根据模型能力显示：

| 场景 | 提示 |
|------|------|
| 上传了 3 张图，模型只支持 1 张 | ⚠️ 黄色提示条："当前模型仅支持 1 张参考图，将以图1 作为底图，其余图片作为风格参考" |
| 模型不支持图生图 | 🔴 红色提示："当前模型不支持图生图，请切换模型" |
| 未上传底图 | "请先上传想要修改的图片" |

## 6. 与现有功能的关系

- **不破坏现有 4 分类参考图**：模式2 保留原有界面和逻辑
- **不破坏现有生图流程**：原有"描述画面 → 配置 → 生成"保持不变
- **新模式作为增强**：可以只填提示词生图，也可以叠加模式1和模式2
