# 视频生成全面重构方案

## 背景

当前视频生成功能存在两大问题：
1. **自由创作**：功能残缺——不支持文生视频、参数不生效、无音频控制
2. **项目模式**：与分镜提示词脱节、无连续性控制、无逐镜头预览/编辑

## 模块A：自由创作界面增强

### 现有问题修复

| 问题 | 修复 |
|------|------|
| 必须上传图片（图生视频） | 支持纯文生视频（无参考图也可生成） |
| 时长/分辨率/模型参数不生效 | `freeVideoGen` API 调用传入真实参数 |
| 只能本地上传图片 | 新增「项目图库」面板，可直接选角色/场景图 |
| 无音频控制 | 添加「生成音频」开关，传 `generate_audio` 参数 |
| 无镜头来源 | 可从项目导入分镜提示词作为 prompt |

### 新增 UI 元素

自由创作标签页下新增「音频生成」开关和「项目素材引用」面板：

```
✏️ 自由创作  |  📂 项目模式

[上传图片区]

[项目素材引用区]  ← 新增：可浏览所有项目的角色/场景图
  ┌──────────────────┐
  │ 项目: 我的项目     │  ← 下拉选择项目
  │ [👤 主角A] [👤 配角B] │  ← 点击选为参考图
  │ [🌆 古堡大厅] [🌆 花园]│
  └──────────────────┘

[prompt 输入区]

模型 | 比例 | 分辨率 | 时长
[生成音频] ☑️  ← 新增开关

[生成视频] 按钮
```

## 模块B：项目视频生成（核心重构）

### B1. 数据流

```
第6阶段（提示词生成）
  ├── 06_提示词/角色提示词.md
  ├── 06_提示词/场景提示词.md
  └── 06_提示词/分镜提示词_第一幕.md  ← 每条提示词对应一个镜头
      06_提示词/分镜提示词_第二幕.md
      ...

第7阶段（视觉素材）
  ├── 07_视觉素材/角色/主角A.png      ← 角色定妆照
  ├── 07_视觉素材/角色/主角B.png
  └── 07_视觉素材/场景/第1幕-古堡大厅.png  ← 场景概念图

第8阶段（视频生成） — 全新界面
  ├── 读取分镜提示词列表
  ├── 匹配对应的角色/场景图
  ├── 逐镜头生成视频
  └── 输出：08_视频/第一幕/第1场_镜头001.mp4
```

### B2. 镜头列表界面

工作区第8阶段不再调用 `redo_phase`，改为展示镜头操作面板：

```
📋 分镜列表

┌─ 第一幕 ──────────────────────────────┐
│ [▶] 镜头1: "特写，主角从阴影中走出…"  │  ← 点击预览prompt
│    状态: ✅ 已生成                     │     [编辑] [重新生成]
│ [▶] 镜头2: "中景，主角走到窗前…"      │
│    状态: ⏳ 生成中                     │
│ [▶] 镜头3: "全景，窗外城市灯光…"       │
│    状态: ❌ 失败: 内容违规            │     [查看原因] [重试]
└───────────────────────────────────────┘

[全部生成] [全部重新生成] [导出片段] [拼接成片]
```

### B3. 连续性策略

```
镜头类型判定（来自分镜信息）：
  ├── 同场景 + 同机位 → 首帧 = 上一镜头尾帧 + 多模态参考
  ├── 同场景 + 切机位 → 多模态参考（跳过首尾帧）
  └── 换场景         → 新场景参考图 + 角色参考图

多模态参考图（每镜头固定传入）：
  1. 角色定妆照（该镜头涉及的角色）
  2. 场景概念图（该镜头所在场景）
  3. 前3个镜头的任意一张成功输出 → 作为画风锚点
```

### B4. 音频方案

```
├── 支持的 API（如 Seedance）
│     └── 开启 generate_audio → 原生同步音频
│     └── prompt 中写入：角色声音描述（如"低沉沙哑的男声"）
│
├── 不支持原生音频的 API（如 Kling）
│     └── UI 提示"该模型不支持原生音频，需后期配音"
│
└── 角色参考音频
      └── 用户可上传某角色的一段.wav作为音色参考
      └── 系统自动提取角色第一段对白作为音频参考素材
```

### B5. 生成流程

```
用户点击「全部生成」
  → 前端建立 WebSocket 连接
  → 后端逐个镜头提交生成任务
  → 每完成一个，推送到前端更新状态
  → 前端实时显示进度条 + 当前镜头

断点续生成：
  ├── 每个镜头生成结果保存在 08_视频/状态.json
  ├── 格式：{ shots: [{ index, status, video_path, error }] }
  ├── 重新生成时跳过已成功的镜头
  └── 失败镜头显示错误原因，可单独重试
```

## 模块C：自由创作↔项目素材互通

### 新增 API

| 端点 | 功能 |
|------|------|
| `GET /api/projects/:name/prompts` | 获取项目第6阶段的分镜提示词列表 |
| `GET /api/projects/:name/visual-assets` | 获取项目的角色/场景图（已有） |
| `POST /api/video-gen/free` | 增强版，支持文生视频+参数+音频 |

### 文件命名规范

保存到项目时，文件名格式：
```
08_视频/
├── 片段/
│   ├── 第一幕_第1场_镜头001.mp4
│   ├── 第一幕_第1场_镜头002.mp4
│   └── 第一幕_第2场_镜头001.mp4
├── 状态.json
└── 成片.mp4
```

## 实施计划

### Phase 1：自由创作增强（前端+后端）
1. 修复 `freeVideoGen` API：传真实模型、分辨率、时长、音频参数
2. 前端支持文生视频（无图片可生成）
3. 前端添加项目图库选择面板
4. 前端添加音频开关

### Phase 2：项目模式新界面（前端）
1. 第8阶段新 UI：分镜提示词列表+逐镜头操作面板
2. 每个镜头可查看/编辑prompt
3. 选择参考图（角色+场景）
4. 批量/单独生成按钮

### Phase 3：后端生成管线（后端）
1. 重写 `VideoProducer` 支持多模态参考
2. 分镜脚本写入机位信息
3. 连续性判定逻辑
4. 断点续生成 + 状态追踪
5. 音频参考传递

### Phase 4：导出与拼接
1. 逐镜头下载
2. 简单拼接成片（现有 VideoConcat）
3. 文件按幕/场/镜头命名
