# 多智能体故事创作系统 — 项目说明

## 一句话定位

输入一个故事想法 → 产出完整剧本 + 角色设定图 + 分镜脚本 + AI 视频提示词 → 一键导出 Word 交稿。

---

## 技术架构

| 层 | 技术 |
|----|------|
| 前端 | React 19 + TypeScript + Three.js（星图背景）|
| 后端 | Python FastAPI + WebSocket |
| 通信 | WebSocket 实时流（不是 HTTP 短轮询）|
| 存储 | 本地文件系统（Markdown + JSON + 图片）|
| AI 接入 | OpenAI 兼容 API + Kling + Runway + Pika + Luma + Seedance |

---

## 六个 Agent 流水线

```
任务指令 → 故事大纲 → 完整剧情 → 完整剧本 → 视觉提取 → 分镜设计 → 生图/视频提示词
```

| 序号 | Agent | 产出 |
|------|-------|------|
| 1 | TaskComposer | 故事设定 + 创作要求 |
| 2 | OutlineWriter | 故事大纲（分幕结构）|
| 3 | PlotWriter | 逐集完整剧情 |
| 4 | ScreenplayWriter | 剧本（对白 + 动作描述）|
| 5 | VisualExtractor | 角色 / 场景 / 道具清单 |
| 6 | Storyboarder | 逐镜分镜脚本 |
| — | PromptEngineer | 生图/视频提示词 |

每个阶段三步骤：**AI 生成 → 审核（AI 自动/用户手动）→ 内容保存 → 进入下一阶段**。

---

## 为什么用 WebSocket

- 每阶段生成可能几分钟，用户需要看到**实时进度**
- token 逐字推前端，不是等好几分钟弹出一大段
- 一条连接贯穿全部阶段，支持中断、修改、重试

---

## 生图系统

| 功能 | 说明 |
|------|------|
| 自由模式 | 输入提示词 → 选模型 → 生成 |
| 项目模式 | 加载剧本 → 自动提取角色/场景 → 批量生成 |
| 角色一致性 | 同一角色的设定图作为参考图传递，跨模型保持长相一致 |
| 支持模型 | OpenAI 兼容聚合平台下所有图像模型 |

---

## 视频生成系统

| 模型 | 后端 | 自动路由 |
|------|------|----------|
| doubao-seedance-* | 火山引擎 Seedance | 默认 |
| kling-v2-master 等 | 可灵 Kling | 模型名含 `kling` |
| gen3-alpha-turbo 等 | Runway Gen-3/4 | 模型名含 `runway`/`gen3`/`gen4` |
| pika-2.2 等 | Pika | 模型名含 `pika` |
| dream-machine 等 | Luma Dream Machine | 模型名含 `luma`/`dream-machine` |
| 其他 | 火山引擎 Seedance | 兜底默认 |

选模型 → 系统自动检测 → 调用对应后端 API → 轮询等结果 → 返回视频 URL。

视频生成页会显示 **"调用: 可灵 Kling"** 标签，如果选了非视频模型会有 **"请确认该模型是否支持视频生成"** 提示。

## Markdown → Word 导出

纯 Python 正则渲染器，零第三方依赖。支持：

- 标题（H1/H2/H3）、加粗、斜体、代码
- 表格、列表、链接
- 单阶段导出、批量合并导出、分集导出、一键全阶段下载

## 市场标准剧本格式（script_format = "2"）

```
第X集
1-1 白天 内 地点
出场人物：A、B

▶ 动作/镜头描写。
A（情绪）："对白"
▶ 动作。
B（情绪）："对白"
```

符合行业短剧剧本规范，剧组拿到直接拍。

## 面试常见问题备忘

**Q: 为什么分六个 Agent 而不是一个？**
A: 每个专注一件事，质量更高。一个 GPT 生 80 集剧本会忘前面写什么。分段生成 + 前序回顾保证连贯。

**Q: 角色一致性怎么保证？**
A: 第一个角色图生成后存为参考图，后续所有生图/生视频接口都带上同一张参考图。换模型角色不变。

**Q: WebSocket vs SSE vs 短轮询？**
A: WebSocket 全程一条连接，token 逐字推。短轮询每 2 秒请求一次服务器扛不住。SSE 只能服务端推，不能客户端中断/修改——这系统需要双向。

**Q: 为什么不用 Next.js/Nuxt/SSR？**
A: 这是单页工具应用，不需要 SEO 和 SSR。Vite + React SPA + FastAPI 后端是最轻快的组合。

**Q: 你最大的收获？**
A: 学会用 AI 辅助把想法变成完整产品。从 prompt 工程到架构决策到验收流程，全链路一个人走完。这种效率是传统手写代码做不到的。
