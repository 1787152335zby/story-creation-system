# 分镜驱动生图（逆推架构） Spec

## Why
当前生图流程：视觉提取阶段全部提取角色/场景/道具 → 用户手动在生图模式挑选 → 生成大量无用图。核心矛盾：
1. 角色和道具各生成一次，但道具本就是角色的配饰/场景的组成部分，分开生成重复浪费
2. 小配角、一闪而过的场景也全部生成，大量算力浪费
3. 同一个角色同一状态在多个镜头出现，每次都重新生成

## Core Idea：分镜需要什么图，就生成什么图
从分镜脚本反向推导生图清单 → 去重 → 按需生成 → 自动引用

### 合并角色+道具为"角色出场状态"
不再单独生成道具图。配饰/道具直接融入角色形象或场景环境：
- **人物出场图**：角色 + 配饰/随身道具 + 变体状态 → 1张图
- **场景环境图**：场景 + 场景专属道具 + 光线/时间/变体 → 1张图
- **关键道具图**：仅独立的线索物品（如录音机、密信）→ 可选

## What Changes

### 后端：新增"生图需求分析"阶段
- 在分镜脚本生成后，新增 `ImageDemandAnalyzer` agent
- 扫描分镜脚本 → 提取每个镜头的角色(含变体)+场景 → 去重 → 输出生图清单 JSON
- 清单格式：`[{type: "character", name: "角色名", variant: "变体名", scene: "所在场景", props: ["配饰1"]}, ...]`
- 有分镜脚本则用精确模式，无分镜脚本 fallback 到当前的 VisualBibleExtractor 全量模式

### 前端：生图模式直接加载清单
- 生图项目模式不再从角色/场景/道具三个列表分别挑选
- 改为直接展示"分镜生图清单"：按角色出场状态列表，每个条目显示关联的镜头
- 点击条目 → 自动使用该角色的提示词 + 场景参考图 → 一键生成
- 已生成的图自动关联回对应镜头

### 兼容性
- **BREAKING**：生图项目模式的交互从"三列表挑选"改为"清单驱动"
- 自由生图模式不变
- 视频模式保持现有分镜选择方式

## Impact
- Affected specs: 视觉提取、生图项目模式、视频项目模式
- Affected code:
  - `agents/image_demand_analyzer.py` — 新增

## ADDED Requirements

### Requirement 1: 分镜驱动的生图需求分析
系统 SHALL 在全部分镜脚本生成完毕后（阶段5后），自动扫描分镜脚本，提取去重后的生图需求清单。

#### Scenario: 短剧3集分镜分析
- **GIVEN** 项目已完成分镜脚本阶段，分镜每镜头标注了出场角色和场景
- **WHEN** 生图需求分析阶段执行
- **THEN** 输出清单包含：每个角色×每个出场状态（服饰/变体）×关联镜头列表
- **AND** 同一角色同一状态在多个镜头出现 → 仅生成一次，关联多个镜头

### Requirement 2: 生图清单展示替代三列表
系统 SHALL 在生图项目模式展示"分镜生图清单"，替代当前的角色/场景/道具三列表。

#### Scenario: 查看生图清单
- **WHEN** 用户进入生图项目模式
- **THEN** 展示角色出场状态列表（按角色分组）
- **AND** 每个条目显示角色名、变体状态、配饰、关联镜头数
- **AND** 场景环境图条目显示场景名、变体状态、关联镜头数

### Requirement 3: 道具融入角色/场景
系统 SHALL 不再独立生成道具图，道具作为配饰融入角色形象或场景环境。

#### Scenario: 角色佩戴指虎
- **GIVEN** 分镜标注角色"林辰"佩戴"指虎"
- **WHEN** 生成该角色出场图
- **THEN** 提示词中包含指虎的外观描述，生成佩戴指虎的角色整体图
- **AND** 不再单独生成"指虎"的道具图
