# 视觉提取数据质量与生图项目模式综合修复 Spec

## Why
用户在测试56项目中发现多处数据质量和生图模式交互问题：

### 数据层
1. **场景缺少层级关联**：学校体育馆（外部）、体育馆擂台（内部）、体育馆上空·黄昏（俯视）三个场景各自独立为 `is_base`，没有父子/关联关系，无法体现空间归属
2. **道具重复**：`围绳`(PROP_008) 和 `擂台围绳`(PROP_003) 本质是同一物品，LLM 提取时未去重
3. **道具缺少细节描写**：`appearance` 字段多为模糊描述（如"普通擂台围绳，可能为红色或蓝色（未明确）"），生图时无法生成准确形象
4. **场景提示词强制"正面视角"**：所有场景 prompt 末尾拼接了"正面视角，从正前方观看"，导致模板生成的鸟瞰图/背面视角也变成正面图

### 交互层
5. **角色无变体层级展开**：基础角色和变体平铺显示为列表，无折叠/展开树结构
6. **参考图强制4分类**：只能将参考图归入画风/人物/场景/道具类别，没有自由参考图模式
7. **项目历史不显示**：项目模式生成的图片不出现下方历史区
8. **部分交互无响应**：勾选框、图片点击放大等操作失效

## What Changes

### 后端
- **场景提取提示增强**：在 visual_bible.py 的提取 prompt 中增加空间归属关系标注指引
- **道具去重**：在道具保存阶段检测同名/近义道具合并
- **移除"正面视角"默认拼接**：场景提示词不再强制加视角描述

### 前端
- **角色/场景树折叠**：ProjectImageGenForm 中基础角色显示 `▸`/`▾` 折叠图标，变体折叠在下方
- **自由参考图模式**：新增"通用参考图"上传区，不强制分类
- **历史区修复**：确保项目模式生成历史正确刷新
- **交互修复**：修复勾选框无响应、参考图点击放大、图片缩略图显示

## Impact
- Affected specs: 视觉提取数据质量、生图项目模式用户体验
- Affected code:
  - `core/visual_bible.py` — 场景提取 prompt
  - `agents/prompt_factory.py` — 场景提示词生成、变体拆分
  - `agents/image_artist.py` — 场景生图视角
  - `src/components/ProjectImageGenForm.tsx` — 角色/场景树、参考图、历史

## MODIFIED Requirements

### Requirement 1: 场景父子关联
系统 SHALL 在视觉提取阶段识别场景的空间包含关系（如学校体育馆包含体育馆擂台），在 JSON 中增加 `parent_scene` 字段。

### Requirement 2: 道具去重
系统 SHALL 在保存道具 JSON 时检测同名/近义道具（如"围绳"与"擂台围绳"），合并为单一道具。

### Requirement 3: 场景提示词移除强制视角
系统 SHALL 在生成场景提示词时仅在用户明确选择视角时才添加视角描述，不再默认拼接"正面视角，从正前方观看"。

### Requirement 4: 角色变体折叠树
ProjectImageGenForm 的基础角色 SHALL 显示可折叠的变体子列表，基础角色和变体不再平铺。

### Requirement 5: 自由参考图模式
系统 SHALL 提供"通用参考图"上传区，用户可将图片作为参考图而不强制归入四大类别。

### Requirement 6: 项目历史修复
项目模式下生成的图片 SHALL 出现在下方历史区。

### Requirement 7: 交互修复
- 勾选框点击 SHALL 正常切换选中状态
- 参考图点击 SHALL 可放大预览
- 已确认的角色/场景图片 SHALL 显示缩略图
